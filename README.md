## Project Scope

The framework automates core Trello functionalities including:

### Board Operations

-   Create board
-   Retrieve board details
-   Update board
-   Delete board

### List Operations

-   Create list within board
-   Retrieve lists
-   Update list
-   Archive/Delete list

### Card Operations

-   Create card within list
-   Retrieve card
-   Update card
-   Delete card

Each test validates: - HTTP status codes - Response body correctness -
JSON path values - Data consistency between dependent API calls

------------------------------------------------------------------------

## Framework Architecture

The project follows a clean and maintainable structure:

-   Centralized request specification configuration
-   Separation between configuration and test logic
-   Modular and reusable components
-   Clear and readable assertions
-   Structured end-to-end workflow scenarios

This design approach improves maintainability, scalability, and
readability.

1.  Clone the repository
2.  Run: mvn clean install
3.  Execute: mvn test

Tests can also be executed directly from the IDE.
